Building the ManagedPython compiler and runtime.

  1. Setup the COM+ 2.0/NGWS environment.  Our makefiles assume that all the NGWS tools are on your path – so please ensure this is true by running the corvars.bat batch file provided by COM+.
  2. Setup the CPython environment – this involves installing the core Python support from http://www.python.org/download/download_windows.html, and also the Python for Windows extensions, build 132 or later, from http://starship.python.net/crew/mhammond/win32 (Specifically, build 132 can be found at http://starship.python.net/crew/mhammond/downloads/win32all-132.exe)
  3. Run nmake from the main src directory.  This will recursively invoke nmake on the relevant sub-directories, and will build the compiler and runtime.  In addition, the built compiler will be used to create the builtin functions and modules (not that we have many yet!)
  4. Run the test suite.   The file p2c\testall.bat is useful for this purpose – it compiles, verifies and executes every test in the suite directory.
  5. Write or run some other code!  The file “make.bat” is useful for this purpose.  E.g., to compile and run mytest.py, you can execute the command make mytest.  This will compile the source file, verify the resulting PE file, and if successful, run the resulting .exe.  Alternatively, you can run cpy.py (the Python Compiler) directly – execute cpy.py with no arguments to see usage information.

The Python for .NET Compiler

Introduction

This document describes using Python for .NET compiler.

The compiler itself is written in Python, and executed using CPython, or good ol’, traditional Python.  The compiler requires the Python COM extensions, and also that some C# “helper” routines be compiled.  All the gory details can be found in readme.txt – for now, we just focus on using the compiler.

The compiler

The Python compiler is cpy.py, and can be found in the compiler directory.  This Python script should be executed using python.exe.  On Windows NT/2000 machines, you can simply execute cpy.py at a command prompt, and if Python is installed correctly the compiler will automatically be executed.

Compiler arguments

The general format of the compiler command-lines is:

cpy.py [/option[:option-val] …] source_file [[/option[:option-val] …] source_file]

Without any options, the compiler will build the source file into an executable.  Thus, the following command:

cpy.py HelloWorld.py

will created HelloWorld.exe in the current directory.

The compiler recognises the following options:

Option

Description

/output-file:string

The file holding the resulting module (and possibly assembly)

/dll

Creates a DLL instead of a .EXE

/debug-info

Emits symbolic debug information, allowing Python code to be debugged in any .NET debugger (including MSVC7)

/source-file:string

The name of the source file to report in the debug information

/verbose-level:int

The verbosity of the compiler output.  Increasing the verbosity level will also include any internal tracebacks generated by the compiler (default=1).  Specifying this option without a value will increase the current level.

/warning-level:int

The warning level (default=2).  Specifying this option without a value will increase the current level.

/module-name:string

The name of the COM+ module generated.  The default is derived from the source file name.

/assembly-desc:string

Description for the generated assembly

/assembly-alias: string

Default alias for the generated assembly

/assembly-keyfile: string

Filename holding the assembly key pair

/assembly-name: string

The assembly name written to the assembly

/assembly-fullname: string

Full name for the generated assembly

/assembly-filename: string

The assembly file name - overrides /output-file

/reference: string

Reference metadata from the named file

/no-python-runtime

Dont add a reference to the standard Python runtime

 

All options can be abbreviated, so long as the option remains unique.  For example:

cpy.py /de /out:test.exe HelloWorld.py

will compile HelloWorld.py to test.exe with debug information.
